## 4. Intended Audience

## 5. Context
What should the council know before reviewing it?

## 6. Specific Questions
List the exact questions you want answered.

Examples:
- Is this differentiated enough?
- Is the audience too broad?
- What is overbuilt?
- What should be cut?
- Would this actually perform as content?
- Is the positioning clear?

## 7. Constraints
Any important constraints?
Examples:
- low budget
- solo founder
- wants to stay simple
- should work as a standalone product first
- should stay narrow in V1

## 8. Requested Review Mode
Choose one exact string:
- `full council`
- `subset review`
- `single-seat review`

Any value other than these three exact strings is invalid and will be rejected.

## 9. Selected Members
- For `full council`: this field must be empty. Listing any members is invalid and will be rejected.
- For `subset review`: list at least two known council member names. Fewer than two is invalid and will be rejected.
- For `single-seat review`: list exactly one known council member name. Any other count is invalid and will be rejected.

## V1 Contract Rules
- Save one artifact per packet file. Multiple artifacts in one packet is invalid.
- All ten fields must be present in this exact order. A file with missing, renamed, reordered, or unknown fields is invalid and will be rejected.
- `artifact_title`, `goal`, `intended_audience`, `context`, and `artifact_text` must be non-empty. An empty value for any of these fields is invalid and will be rejected.
- `specific_questions` must contain at least one item. An empty list is invalid and will be rejected.
- `constraints` must contain at least one item. An empty list is invalid and will be rejected.
- `artifact_text` must contain the complete artifact. A truncated or placeholder value is invalid.
